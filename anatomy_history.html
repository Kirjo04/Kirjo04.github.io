Anatomy of a URL

Scheme (Protocol)

The scheme, or protocol, is the part of a URL that begins with “http” or “https” and defines how data is transferred between a browser and a server. HTTP (Hypertext Transfer Protocol) was created by Tim Berners-Lee in 1992 as a simple, text-based system, while HTTPS adds encryption for secure communication. Earlier protocols like FTP and Gopher handled file transfers and metadata, but HTTPS has become the modern standard due to its security.

Authority (Domain and Port)

The authority includes the domain and optional port, identifying where a resource is located. Domains were introduced to replace a single, unscalable host file with a hierarchical system, allowing users to query only the part they need; this system also introduced top-level domains like .com, .edu, and .org. The “www” subdomain was originally used to distinguish web services from others, but today it is mostly a convention, and port numbers are typically omitted when using standard web ports.

Path

The path specifies the exact location of a resource within a website, guiding the browser to a specific page beyond the homepage. It evolved from early systems like Multics, which used different symbols before adopting the forward slash (/) format still used today. This structure allows websites to organize and access content in a clear, hierarchical way.

Parameters

Parameters are optional parts of a URL used to pass data, often for filtering content or tracking user activity. They originated from early web features like the "isindex" tag, which allowed users to search page indexes directly. Over time, parameters became more flexible and widely used for dynamic content and analytics.

Anchor

Anchors, or fragments, link to a specific section within a webpage rather than loading a new page. They improve navigation, especially for long documents, by jumping directly to relevant content. Unlike other parts of a URL, anchors are handled entirely by the browser and are not sent to the server.
