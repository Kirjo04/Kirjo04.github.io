What is a URL?

Creation

Tim Berners-Lee created HTTP, HTML, and URL in 1992.

Syntax

  • :// - goes between the protocol and domain
  • / - separates the path
  • ? - denotes the start of the parameters
  • # - indicates an anchor

Problems

  • 404 Not Found - the page no longer exists
  • URL does not go to the correct webpage
  • Duplicate content - caused by extra slashes or incorrect parameter use

Future

URLs will probably become obsolete in the near future with the rise of AI search

Structure Components

Scheme/Protocol (http, https, ftp) — Defines how data is transferred between browser and server. HTTPS is secure (encrypted).

Domain Name (example.com) — Human-readable address that maps to an IP address.

Subdomain (www., blog., store.) — Prefix that divides sections of a website.

Path (/folder/page) — Points to a specific resource or page within a website.

Query String (?key=value&key2=value2) — Sends data to the server (used for searches, filters, tracking).

Fragment/Anchor (#section) — Jumps to a specific part of a webpage.

DNS & Resolution

DNS (Domain Name System) — Translates domain names into IP addresses.

IP Address — Numeric address of a server (e.g. 192.168.1.1).

Caching — Browsers store DNS results to load sites faster.

Encoding & Rules

URL Encoding — Converts special characters into safe formats (e.g. space → %20).

Case Sensitivity — Domain names are not case-sensitive, but paths sometimes are.

Reserved Characters — Some symbols have special meanings like ?, #, &, /.

Performance & SEO

Clean URLs — Short, readable URLs improve usability and SEO.

Canonical URL — Preferred version of a webpage to avoid duplicate content issues.

Slug — The readable part of a URL (e.g. /best-games-2026).

How URLs are Used

Navigation between pages

Sharing resources online

API requests in web development

Tracking user behavior (UTM parameters)

How they work?

  1. User enters URL → Browser queries DNS
  2. DNS returns IP address
  3. Browser sends request to server
  4. Server sends content
  5. Browser displays content

Role

To create a user friendly "code" for locating resources on the internet
